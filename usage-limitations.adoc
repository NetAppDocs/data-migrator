= Usage limitations
:icons: font
:imagesdir: ./media/

[.lead]
Understanding the usage limitations and server types supported by NetApp Data Migrator (NDM) can help to make your data migration activities more efficient.

*Summary of file server types and protocols supported by NDM*

|===
| Category | Supported

| Source file server	 | Dell Isilon, ONTAP, Vanilla Linux and Windows servers
| Destination file server | Any service level of ANF/GCNV
| NFS protocol version| 3.0, 4.1
| SMB protocol version| 2.0, 3.0, 3.1

|===

*Discovery and migration limitations*

|===
| Category | Limitation

| Number of export paths that can
run simultaneously	 | <tbd>
| Maximum number of files in a directory
for NFS	 | 1 million
| Maximum number of files in a directory
for SMB | 1 million
| Maximum depth| 75
| Number of workers| <tbd>
| Worker sizing| <tbd>
|===

*Limitation: NFS v4.1 in multi-worker scenarios*

Due to restrictions on the number of concurrent mounts, NFS v4.1 is currently de-prioritized for use in multi-worker scenarios. Where multiple workers attempt to mount the same NFS v4.1 share simultaneously, issues may arise that impact workflow stability and performance.

Temporary guidance:
Avoid using NFS v4.1 for workflows that involve multiple workers accessing the same share. It is recommended to use NFS v3.0, or other configurations that support parallel access for multi-worker environments.

*Other limitations*
|===
| Category | Limitation

| Network accessibility | It is recommended to use NDM in  private networks
| Directory level migration | NDM does not support Directory level migration.
| Domain support | No domain support, accessible via IP address only
| Security | NDM uses self-signed SSL certificate
| User interface | Optimized for 1920 x 1080 screen resolution. Mobile displays are not supported.
| Windows worker deployment | Windows worker must be part of the same root domain as the destination
| Performance | NDM application performance is dependent on source and destination network throughput
| Protocol migration - Version | Cross-version migration within the same protocol is not supported (for example, NFS v3 to NFS v4)
| Protocol migration - Type | Cross-protocol migration is not supported (for example, NFS to SMB)
| File names | NDM cannot migrate system-generated files owned by source 

|===
