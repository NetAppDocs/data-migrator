= NDM usage limits
:icons: font
:imagesdir: ./media/

[.lead]
Understand the usage limits of NetApp Data Migrator (NDM) to help make your migration activities more efficient.


*Discovery and migration limits*

|===
| Category | NFS | SMB 

| Number of export paths that can
run simultaneously	 | 4  | 2
| Maximum number of files in a directory | 1 million | 1 million
| Maximum depth| 75 | 75
| Number of workers| NFS3: 4;  NFSv4.1: 1 | 1 
| Worker sizing| NFS: 8 core CPU, 32GB memory | 16 core CPU, 64GB memory
|===


*Other usage limits*
|===
| Category | Limits

| Network accessibility | It is recommended to use NDM in  private networks
| Directory level migration | NDM does not support Directory level migration.
| Domain support | No domain support, accessible via IP address only
| Security | NDM uses self-signed SSL certificate
| User interface | Optimized for 1920 x 1080 screen resolution. Mobile displays are not supported.
| Windows worker deployment | Windows worker must be part of the same root domain as the destination
| Protocol migration - Version | Cross-version migration within the same protocol is not supported (for example, NFS v3 to NFS v4)
| Protocol migration - Type | Cross-protocol migration is not supported (for example, NFS to SMB)
| File names | NDM cannot migrate system-generated files owned by source 

|===
