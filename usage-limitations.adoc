= NDM usage limits
:icons: font
:imagesdir: ./media/

[.lead]
Understand the usage limits of NetApp Data Migrator (NDM) to help make your migration activities more efficient.


*Discovery and migration limits*

|===
| Category | NFS | SMB 

| Number of export paths that can
run simultaneously	 | 4  | 2
| Maximum number of files in a directory | 1 million | 1 million
| Maximum number of workers| 4 | 4
| Worker sizing| 4 core CPU, 16 GB memory | 16 core CPU, 64 GB memory
| Control plane sizing| 16 core CPU, 64 GB memory | 16 core CPU, 64 GB memory 
|===


*Other usage limits*

[cols="30,70"]
|===
| Category | Limits

| Active destination support
| NDM does not support an active destination (when a target storage is actively used or written before Cutover).
| Case sensitive files 
| For SMB, NDM migrates only one of the case-differing files created using NFS in a folder and errors out the other file. This occurs because SMB can’t accept both files. You can run discovery using NFS to identify these case-sensitive files.
| Directory level migration | NDM supports migration of entire export paths only. It does not support migrating individual directories within an export path.
| Network accessibility | NetApp recommends using NDM in private networks.
| NFSv4 ACLs | NDM doesn’t stamp access control lists (ACLs) with NFSv4, it only applies basic permissions in the destination. This behavior is similar to NFSv3.
| Protocol migration - Type | Cross-protocol migration is not supported, for example, NFS to SMB.
| Protocol migration - Version | Cross-version migration within the same protocol is not supported, for example, NFSv3 to NFSv4.
| SMB permissions
| Migration of SMB SACL (audit permissions) is not supported.
|  SMB files with trailing spaces  
| When migrating over SMB, NDM errors out files with names that contain trailing spaces because SMB does not permit these file names. 
| SMB special files | NDM discovers redirects (symbolic links, hard links, junction points, Alternate Data Streams (ADS), and volume mount points) and reports them after discovery. Migration of ADS, sparse files, and SMB redirects is not supported.
| Security | NDM uses self-signed certificates to encrypt web traffic with SSL/TLS.
| Sparse files | Sparse files become full-size files when they migrate, NDM doesn't preserve sparsity. You need extra storage for these files.
| System files | NDM cannot migrate system-generated files that the source owns.
| User interface | NDM is optimized for Google Chrome and Firefox browsers using 1920 x 1080 screen resolution; mobile displays are not supported.
| Windows worker deployment | The Windows worker must be part of the same root domain as the destination.

|===
