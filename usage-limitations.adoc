= NDM usage limits
:icons: font
:imagesdir: ./media/

[.lead]
Understand the usage limits of NetApp Data Migrator (NDM) to help make your migration activities more efficient.


*Discovery and migration limits*

|===
| Category | NFS | SMB 

| Number of export paths that can
run simultaneously	 | 4  | 2
| Maximum number of files in a directory | 1 million | 1 million
| Maximum number of workers| NFSv3: 4;  NFSv4.1: 1 | 4
| Worker sizing| 4 core CPU, 16 GB memory | 16 core CPU, 64 GB memory
| Control plane sizing| 16 core CPU, 64 GB memory | 16 core CPU, 64 GB memory 
|===


*Other usage limits*
|===
| Category | Limits

| Active destination support
| NDM does not support an active destination (when a target storage is actively used or written before Cutover).
| Case sensitive files 
| For SMB, NDM migrates only one of the case-differing files created using NFS in a folder and errors out the other. This occurs because SMB can’t accept both files. NetApp recommends running discovery using NFS to identify these case-sensitive files.
| Directory level migration | NDM supports migration of entire export paths only. It does not support migrating individual directories within an export path.
| Domain support | There is no domain support, NDM is accessible using an IP address only.
| Network accessibility | NetApp recommends to use NDM in  private networks.
| Protocol migration - Type | Cross-protocol migration is not supported (for example, NFS to SMB).
| Protocol migration - Version | Cross-version migration within the same protocol is not supported (for example, NFS v3 to NFS v4).
| SMB permissions
| Migration of SMB SACL (audit permissions) is currently not supported
| SMB special files
| NDM discovers redirects (symbolic links, hard links, junction points, and volume mount points) and reports them after discovery. Migration of Alternate Data Streams (ADS), sparse files, and SMB redirects is currently not supported.
| Security | NDM uses self-signed certificates to encrypt web traffic with SSL/TLS.
| System files | NDM cannot migrate system-generated files owned by the source.
| User interface | NDM is optimized for Google Chrome and Firefox browsers using 1920 x 1080 screen resolution; mobile displays are not supported.
| Windows worker deployment | Windows worker must be part of the same root domain as the destination.
|===
