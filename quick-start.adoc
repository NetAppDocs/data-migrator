= Quick start for NetApp Data Migrator
:icons: font
:imagesdir: ./media/

[.lead]
To get started with NetApp Data Migrator, you need to set up control plane and worker virtual machines (VMs) using either AWS, Azure, or Google Cloud services, or Open Virtual Appliance (OVA) templates. Then, define your credentials and create your first project.


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One] Install NetApp Data Migrator

[role="quick-margin-para"]
Depending on your environment, you install NetApp Data Migrator using AWS, Azure, or Google Cloud services, or OVA templates in a VMWare vSphere environment.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two] Log in to NetApp Data Migrator

[role="quick-margin-para"]

You acquire the username and default password, log in as an administrator using the default username and password, and change your login credentials.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png[Three] Create your projects

// This step shows an example of using a list under the block title, rather than a paragraph. Note that lists are styled with the `quick-margin-list` role attribute to ensure correct indentation.

You use the Projects tab in the NetApp Data Migrator control plane to create, edit, and switch between projects and workers.

.What's next?

After you create your projects, you can add and manage users and file servers, configure real-time notifications, set migration options, and manage your jobs and job runs.
