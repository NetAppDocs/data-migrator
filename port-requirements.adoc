= Port requirements for NetApp Data Migrator
:icons: font
:imagesdir: ./media/

[.lead]
Ensure that certain TCP ports are open to allow communication between the control plane and worker VMs in NetApp Data Migrator.

The following tables list the ports that the control plane and worker virtual machines use to communicate.

*Control plane*

[cols="1,1,1,1"]
|===
|Service | TCP port |  From | To

| Temporal Server | 7233 | Workers and clients | 
| Temporal Server | 7234 || Internode connection, if using clusters
| Temporal UI | 8080  | Default port, if enabled | 
| PostgreSQL (Temporal database) | 5432 |Temporal services | 
| Redis | 6379 |Workers or control plane services |  
| API or UI ingress (your application) | 80 or 443, as applicable  | |	
| Grafana | 3000 | |	
|===

*Workers*

[cols="1,1,1,1"]
|===
|Service |TCP port |  From | To

|Outbound to control plane | 7233 | | Temporal frontend
|Outbound to control plane | 6379 | | Redis
|Outbound to control plane | 80 or 443 | | API or UI, if they call it
|Outbound to storage |NFS: 2049 | | NFS servers 

*Note*: Include TCP or UDP port 111 if you need the portmapper service for Remote Procedure Calls.
|Outbound to storage | SMB: 445  | | SMB servers
|===
